Agent Office Suite, also called AOSE, has been added to the Open Orchestrators directory. The Apache-2.0 project describes itself as an office suite where AI agents are first-class collaborators across docs, databases, slides, and flowcharts.

This matters for Open Orchestrators readers because AOSE puts the agent inside shared work artifacts instead of keeping it in a separate chat surface. The public README describes `@agent` comments, contextual tasks, MCP tools, self-registration, approval-based activation, version snapshots before agent edits, and reversible change history.

For builders and operators, the practical signal is that AOSE treats agent work as something that needs a durable office surface: humans and agents work on the same document, table, presentation, or diagram, while edits remain traceable and recoverable. The current quickstart uses `npx aose-main`, and the latest checked release is `v2.0.8`.
